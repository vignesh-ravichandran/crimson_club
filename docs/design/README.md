# Design docs

- **UI/UX** decisions and rationale
- **Screens** and layouts (described or linked from Figma)
- **Components** and design system notes (captured as we develop code)

Design is captured **as we build** — when you add or change a screen or component, update or add a doc here so design and code stay one source of truth.

## Naming

- `screens-<feature>.md` — Screens and layouts
- `components.md` or `components-<area>.md` — Component inventory and usage
- `design-decisions.md` — ADRs or notable UI/UX decisions

- **Index:** [INDEX.md](INDEX.md) — list of all design docs (UX blueprint, colour palette, logo) for reference.
