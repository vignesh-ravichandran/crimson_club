# High-level design (HLD)

**Reference documentation** — system-level view: architecture, boundaries, main components, and how they connect.

- System/application architecture
- Service boundaries and responsibilities
- Key integration points and data flow at a high level

## Naming

- `architecture.md` — Overall system or app architecture
- `hld-<area>.md` — HLD for a specific area or service

- **Index:** [INDEX.md](INDEX.md) — list of HLD docs for reference.

Update when architecture or boundaries change.
