# Low-level design (LLD)

**Reference documentation** — detailed design: modules, components, APIs, and behaviour below the HLD.

- Module/component breakdown
- API contracts and behaviour
- Key algorithms or flows (as reference)

## Naming

- `lld-<feature-or-module>.md` — LLD for a feature or module
- `apis.md` or `apis-<area>.md` — API reference or summary

- **Index:** [INDEX.md](INDEX.md) — list of LLD docs (APIs/modules, frontend design) for reference.

Update when you change module boundaries, APIs, or detailed behaviour.
