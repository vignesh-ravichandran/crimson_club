# Tech stack

**Reference** — languages, frameworks, libraries, and tooling we use. Single place to see what we build with.

- Languages and runtimes
- Frameworks and core libraries
- Key tooling (build, test, lint, etc.)

Update whenever we add or change a major technology. Keep it accurate so onboarding and decisions stay aligned.

## Naming

- `tech-stack.md` — Main list (or one file per area: frontend, backend, infra)
- `tech-stack-<area>.md` — Optional breakdown (e.g. `tech-stack-frontend.md`)

- **Index:** [INDEX.md](INDEX.md) — list of tech stack docs for reference.
